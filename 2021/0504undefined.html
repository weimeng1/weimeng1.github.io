hexo三平台（gitee、github、coding）自动一键部署

脚本
发布日期:   2021-05-04
阅读时长:   3 分

事情缘由

最近放假没啥事就把博客微调了一下然后部署到了三个平台上分别是：gitee、github、coding就发现了一个事这个gitee他不能自动部署，只能将文件推送上去之后手动自己点击部署。没办法开头也就忍了，之后有一次在那改着改着特别烦，一直hexo clean 、hexo g 、hexo d。

这样去部署我就下定决心一定要吧这个问题解决。话不多说上主菜。

思路

1.首先开头是找了找有没有这种一键部署gitee的软件，果不其然居然在csdn里给我找到了。首先要先感谢一下这位作者峡州仙士(我的亲情链接里也挂了有兴趣的可以去膜拜一下)。因为他博客的程序链接挂了，特意加他Q问他要。程序链接密码:i393，程序配置方法作者博客里有自行查找修改。

因为文件体积原因我把文件拆分了下载之后把exe文件添加至文件夹即可。

2.接着我又去找了找有没有一键部署hexo的办法不需要一直输入命令。结果还就真找到了（原文章）办法是在博客文件目录下新建一个bat文件代码如下:

@echo off
E: # 修改为你博客所在的盘
cd E:\Blog\blog5 # 修改为你博客所在的文件夹
hexo clean&&hexo g&&hexo d

这个代码要稍作修改别拿着就直接用了。

改进

有了以上两个程序我们就可以很轻松的部署了，这样一来话也就要两步第一步先执行bat命令让代码推送上去接着执行exe程序单独部署gitee。但是其实接触过bat命令的同学应该都知道，bat可以做很多事情例如打开一个程序。我的想法就是这样：

修改bat文件让他 —— 执行部署 —— 然后打开这个exe文件结果就是这样
@echo off
E:
cd E:\Blog\blog5
hexo clean&&hexo g&&hexo d&&start ""  D:\BOLG\hexoblog\"gitee.exe" # 指定你那个exe文件的位置

这串代码看似没有问题但是我也不知道什么原因导致闪退

我想了半天最后采取了看似最蠢也最好用的一种方法那就是套娃，想法如下：

我新建了一个bat文件来打开exe文件然后整体运行起来就是这样

首先先打开第一个bat文件部署完之后第一个bat文件会自动打开第二个bat文件接着第二个bat文件会自动打开那个exe文件。这是最后成品代码记得自己修改路径。

# 第一个bat文件
@echo off
D: #修改为你博客所在的磁盘
cd D:\BOLG\hexoblog\ # 修改为你博客所在位置
hexo clean&&hexo g&&hexo d&&D:&&start ""  D:\BOLG\hexoblog\"gitee.bat" # 修改为你第二个bat文件的位置
# 第二个bat文件
D: # 修改为你博客所在的磁盘
cd D:\BOLG\gitee 你exe文件存放的文件架
start D:\BOLG\gitee\gitee.exe 修改为你exe文件存放的位置。

这条一来的话就大功告成了。
