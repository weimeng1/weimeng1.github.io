0基础教你用hexo+coding搭建博客及美化

个人博客搭建
发布日期:   2020-04-29

博客搭建

说明：

你们可能会问这么多可以部署的地方比如什么国外github还有什么国内的码云啥的为什么会选coding，原因很简单这几个我都试过第一个用的就是github，github如果你部署在这个网站的话访问特别慢这也就是我不选github的原因，当然你如果对其他平台有意见就是想用github的话可以去网上搜一下CDN加速，这个我也不太明白，在这我也就不多赘述了。那回归正题那我不选github为什么不选码云，码云的话这个我也是用过的，这个用的话反正容易出问题我就出现了无法显示样式的问题，所以我后来选择了coding。

需要准备的东西：

  • nodejs
  • nodejs安装教程
  • git
  • git安装教程
  • VScode

    注册coding搭建仓库

1.进入coding官网注册然后登录
2.点击头像左边的第二图标新建一个项目，选择DevOps项目，勾选上Git仓库下方的启用 README.md 文件初始化项目其他选项随便填写然后完成创建

利用脚本快速搭建博客

我们需要选择我们博客要储存的位置新建一个文件随便取一个名字，接着我们打开那个新建的文件右键打开Git bash Here我们输入这样一段代码： 

curl -O https://cdn.jsdelivr.net/gh/kjhuanhao/hexo-script@master/install.sh

它就会为你安装一个install.sh脚本然后我们就开始初始化博客我们继续输入：

source install.sh hexo_win

然后我们要做的只有等待安装完成你就会有一个博客了
安装完成后新建的那个文件夹会有一个hexoblog的文件夹我们先关闭Git bash Here然后到hexoblog文件夹再打开Git bash Here输入这样一段命令：

  hexo s

他会反馈给你这样一个画面
最后一行有一个 http://localhost:4000 我们复制然后到浏览器打开他就会有一个这个画面

PS：这里有一个小细节不要到Git bash Here里选中然后ctrl+c在Git bash Here里ctrl+c是类似退出的意思，我们可以到Git bash Here里选中右键选择copy，这个copy就是复制的意思。

生成一SSH个密钥

1.我们到我们之前存放install.sh脚本的文件夹里然后右键打开Git bash Here运行这样一句话： 

./install.sh git_ssh

运行完毕后会有这样几个选项

(1)Key
(2)SSH
(3)Verify-github
(4)Verify-coding

  • (1)Key 执行1后可以一键生成秘钥如果你的电脑已经存在秘钥,程序将会自动退出

    • (2))SSH 执行2后可以查看你电脑中已生成的秘钥]

    • (3)Verify-github 执行3后可以检查秘钥是否配置github成功

    • (4)Verify-coding 执行4后可以检查秘钥是否配置coding成功

      我们执行命令1创建一个SSH密钥ps：（创建密钥时会要填一些资料你如果不知道填啥的话把那些copy到谷歌翻译里就可以了）
      创建完毕之后可以再执行一次./install.sh git_ssh然后就会有一大段以ssh开头以你的邮箱结尾的你把他们全部选中然后copy

      把密钥添加到你的coding账号中

1.我们打开coding官网然后到右上角头像那里有个个人设置点开右边选项有个SSH公钥
2.点开后在中间上方区域有个新增公钥点开之后有个公钥内容你直接把你copy的那一段ssh公钥粘贴上去公钥名字可以不要填然后到下面有个永久添加你勾选一下然后再点击添加就行了

部署到coding

1.我们打开coding官网然后到头像那里有个团队管理你到那里实名认证一下，接着我们再到之前创建好的仓库在左边有一个代码仓库点开右上角有个克隆点击他，他就会弹出一个框框在左边有个HTTPS或者是SSH我们选择SSH就行了然后复制右边以git开头的一段类似代码之类的东西你先把它复制，打开持续部署再打开静态网站网站名称随便填一下就可以然后保存一下就行了然后你就可以看到右边有一个链接我们先不管他我们接着下面操作
2.然后再到你储存博客的文件夹然后打开hexoblog里面有一个_config.yml的文件右键他然后打开方式我们选择我们之前安装的VScode打开后我们到最下面最后一段我们把它选中删除把下面那一段复制上去然后再repo：后面添加你刚刚复制的那一段以git开头的代码粘贴上去然后保存。

deploy:    
  type: git    
  repo: 这里填你的仓库地址,建议使用SSH地址,它是以git开头的  
  branch: master

然后我们保存然后到再回到hexoblog文件夹右键打开Git bash Here执行 hexo clean 然后执行 hexo g 最后执行hexo d 你的博客就成功部署到coding里了这时候我们就可以打开在静态网站那里看见的那个链接你就可以看见你的博客了。
